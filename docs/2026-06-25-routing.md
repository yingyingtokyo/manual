## Parameter files
### flow direction files
USE QGIS GRASS to create a flow direction map based on USGS DEM (https://apps.nationalmap.gov/downloader/).<br>
![Alt text](image-1.png)

The element at first row and first column (1,1) should be the **west-most and north-most** of the array. We can directly apply QGIS generated flow direction map here.

- xllcorner, yllcorner: XC and YC is the **west-most longitude** and **south-most latitude**.<br>
  ```FORTRAN
    ILOC=XC + (ICOL-II)*SIZE + SIZE/2.0
    JLOC=YC + JJ*SIZE - SIZE/2.0
  ```

- F8 definition in VIC-Res. VIC-Res read flow-direction array from right-to-left, bottom-to-top, and I is column and J is row, so the definition is very different.

    | VIC-Res | GRASS | Direction |
    |---|---|----|
    | 1 | 2 | N  |
    | 2 | 1 | NE |
    | 3 | 8 | E  |
    | 4 | 7 | SE |
    | 5 | 6 | S  |
    | 6 | 5 | SW |
    | 7 | 4 | W  |
    | 8 | 3 | NW |

We should pay extra attention on **column and row** when preparing the flow direction data.

The reading of flow direction data from txt to fortran array is **bottom-to-top, rigth-to-left**. So the last row will be assigned with 1th row and last column will be assigned with 1th column in H(I,J). Here I is column index and J is row index.

### Unit hydrograph file
The unit hydrograph file describes how runoff from a grid cell or a catchment is distributed over time after a unit of rainfall (e.g., 1mm) rainfall. This helps simulate how to generate streamflow that travels through the catchment.

It has two columns (which need to be calibrated later). The first column is the value for lag step $k$ , the second column is the UH weight $w$. Steeper channels are typically represented by smaller $k$ and larger $w$.
- $𝑘$ = coefficient for travel time.
- $w$ = weight of how much storage depends on inflow.
- $Q(t)=\sum _{k=0} ^{nrows-1} w(k)R(t-k)$, where $R(t-k)$=runoff generated at $k$ steps earlier. $w(k)$=UH weight at time lag $k$.
- Example: /home/fs01/yl3984/VICRes/routing/parameters/unit_hydrograph/UH.all represents $Q(t)=R(t)$, which means no streamflow delay.

### Grid location files
- grid_lat.txt
- grid_lon.txt

If we **turn on** the option of reading grid location files, it will read latitude and longitude from these two txt files. The latitude is from north to south for each row, longtitude is from west to east for each column.

If we **turn off** the option of reading grid location files, the locations for upstream grids will be calculated within VIC-Res by using xllcorner, yllcorner, size, nrows and ncols in flow direction file.

### Reservoir files
- reservoirlocation.txt <br>
Use the same size matrix as flow direction files. And the values in the matrix represents index of different reservior. The descritative parameters for dam is from GRanD_reservoirs_v1.3 dataset.

- res.txt <br>
Usually choose 1 (basic) or 3 (when we have observed storages) for operation strategy.
    ```bash
    Hmax(M) Hmin(M) Slive(1000M3)   Sdead(1000M3)   Hturbine(M)     Qdesign (M3/s)  Year    Sinitial(1000M3)        Name
    545     510     444000  88000   216     125     1999    300000  ATYh
    SEEPAGE INFILTRATION
    0.000000        0.000000
    IRRIGATION
    0
    DUMMY
    HYDROPOWER
    1
    ENVIRONMENTAL FLOW
    0.05
    OPERATION STRATEGY
    1
    545     510     150     300
    BATHYMETRY PARAMETERS
    0
    135.186458      0.033764077     -0.00041473     1.11114436
    ```
- Hmax and Hmin<br>
Hmax = ELEV_MASK - DEPTH_M + DAM_HGT_M<br>
Hmin = Hmax - DAM_HGT_M*CAP_REP/CAP_MAX  <br>
- Slive<br>
  Since unit of CAP_RAP is $10^6 m^3$,<br> 
  Slive = (CAP_RAP) * 1000 * 0.9 <br>
- Sdead<br>
Sdead = (CAP_RAP) * 1000 * 0.1 <br>
Total_storage = Slive + Sdead
- Htrubine<br>
Hturbine = (Hmax + Hmin)/2<br>
- Qdesign<br>
Qdesign = 1.3*DIS_AVG_LS/1000 <br>
convert unit from $l/s$ to $m^3/s$<br>
The **coefficient (1.3)** for Qdesign can be changed based on the different levels between peak flow and average flow.
- Year<br>
YEAR from GRanD<br>
- Sinitial<br>
0.5*CAP_MCM<br>
- Environmental flow<br>
0.05*DIS_AVG_LS<br>
- bathymetry parameters<br>
  0: use defaulted bathymetry parameters <br>
  1: need to provide BTHMET_A, BTHMET_B, BTHMET_C, BTHMET_D <br>
  $H(V) = \frac{A}{e^{-BV}+CV+D}$<br>
  $H(0) = A/(1+D)$
- operation strategy<br>
    - RULE1: simplest<br>

    **[Hmax, Hmin, the_day_with_smallest_storage, the_day_with_largest_storage]**<br>

    When the season with largest precipitation, the dams will be decided with smallest storage.

    - RULE2: average monthly observed water level (m)<br>
      - Observed water storage from ResOpsUS or satellite product
      - AREA_SKM from GRanD database
      - Water_level = Obs_water_storage / AREA_SKM + Hmin
    - RULE3: Simulate operation based on water demand.
    - RULE4: Use a predefined release time series data for simulation. 
    - RULE5: Use monthly operating rules that specify release or storage behavior.
    - RULE6: Use a predefined daily storage time series as target.

### Observed discharge files for parameter calibration
- stations.txt
It shows the location information of calibrated grids. It has five columns: NR(I),NAME(I),PI(I),PJ(I),AREA(I).
  - NR: 0/1 (0: disactive, 1: active)
  - NAME: stations name
  - PI, PJ: location of current station (column, row)<br>
    ```FORTRAN
    PI(I)=ICOL+1-PI(I)  !note, the arrays are flipped left to right<br>
    ```
    When reading flowdirection file, **its column and row are flipped**, therefore here **I (column) is the same order as python array, but J (row) should be flipped**.<br>
    For example, if the location of station in flow direction array is (1,1) (column, row), it will be shown as (1,5) in stations.txt.
  - AREA: area of current station
- Simulation steps:
  - we will select the cells and reservoirs that contributes to station.
  - we use MAKE_CONVOLUTIONRS to calculate the flow generation for the required station by using Convolution Method.
  - Finally we will get the flow at all stations in output file.
